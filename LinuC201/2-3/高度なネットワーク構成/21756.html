設定されているルーティングテーブルから、あるホスト 192.168.2.10 への経路を削除したい。
実行するべきコマンドとして最も適切なものはどれか、選びなさい。

正解

route del -host 192.168.2.10

解説

routeコマンドの削除の書式は以下のとおりです。

route del [-host|-net] 宛先アドレス [gw ゲートウェイのIPアドレス]
削除の場合は、識別が可能な場合は宛先アドレスだけでの削除も可能です。

追加・削除の場合の宛先アドレスは、以下の形式等が可能です。
IPアドレス/プレフィックス
IPアドレス netmask サブネットマスク

ここでは経路を削除するための書式である、
route del [-host|-net] 宛先アドレス [gw ゲートウェイのIPアドレス]
が問題になっています。

このような書式となっている選択肢は
・route del -host 192.168.2.10
のみですので、これが正解だとわかります。
このようにはじめの del という指定はハイフン (-) なしで、続く -net または -host はこの通りハイフンをつけて、それぞれ指定する必要があります。

その他の選択肢について解説します。

・route remove -host 192.168.2.10
del ではなく remove と指定してしまっている点が誤りです。このようなキーワードはなく、受け付けられません。

・route -del host 192.168.2.10
del の前にハイフンをつけてしまっている点、host の前にハイフンがついていない点が誤りです。
正しくは del -host と host の方にハイフンをつけます。

・route -remove host 192.168.2.10
-remove という不正な指定を行っている点も、host とハイフンをつけずに書いてしまっている点も誤りです。

【ルーティング関連のコマンド】
パケット転送において、パケットは多くの中継装置（ルータ）によって対象ホストまで運ばれていきます。
その中継の際、目的のホストにパケットを届けるには次にどこへ向かってパケットを転送すればよいかという経路の判断が必要になります。
その判断を行い、適切な経路へパケットを送信することをルーティングといい、そのための情報をルーティングテーブルといいます。
な お、ルータのルーティングテーブルもホストのルーティングテーブルも基本的には同じものですが、ルーターは原則的には自分あてのパケットを受け取ったり自 分からパケットを投げたりせず、中継に徹しますが（実際にはルーティング制御用の様々なパケットを送受信します）、LAN内ホストの場合は、自身のパケッ トをどこに送信するか、という意味合いの経路表のことをルーティングテーブル、といっています。
ルーティングテーブルを構成する各々のルートに は、スタティックとダイナミックという2通りの登録方法がありますが、もっとも単純に、現在のホスト（小規模なルータでも用いられる）がどうルーティング するかを、手動でルーティングテーブルにインストールするのが、スタティックルートです。
一方、大きなネットワークのルーターのルーティングに用いられるダイナミックルーティングでは、ホップ数や遅延といった様々な要素を考慮してメトリックを自動的に計算し、ルーティングを決定します。（詳細は試験範囲外のため、割愛します。）

以下は、ホストのルーティングテーブルにおけるスタティックルートでの、ルーティングテーブルの操作についての説明です。
ルーティングテーブルに関する操作を行うには route コマンド、またはip routeコマンドを使います。

・routeコマンド
route コマンドの使い方を説明します。

表示
route

追加
route add [-host|-net] 宛先アドレス gw ゲートウェイのIPアドレス

削除
route del [-host|-net] 宛先アドレス [gw ゲートウェイのIPアドレス]

追加・削除の場合の宛先アドレスは、以下の形式等が可能です。
IPアドレス/プレフィックス
IPアドレス netmask サブネットマスク

デフォルトゲートウェイ設定
route add default gw デフォルトゲートウェイのIPアドレス

引数を書かなければルーティングテーブルの表示を行う動作になります。
そして、第一引数を add や del とすれば新しい経路を追加・削除できます。
追加や削除の際には追加する経路がホスト単位なのか (-host)、サブネット単位なのか (-net) も指定します。
追加の場合は、最低限、ゲートウェイアドレスと、プレフィックスまたはサブネットマスクを指定します。

削除の場合は、識別が可能な場合は宛先アドレスだけでの削除も可能です。

最後に、少し特別な指定の仕方として、デフォルトゲートウェイの設定方法があります。
この場合 -host や -net とそれに続く 宛先アドレスは指定せず、default というキーワードを代わりに書きます。

以上のように route コマンドはやや複雑な書式になっています。それぞれの書式を正しく読み書きできるようにしておきましょう。

なお、ディストリビューションによっては様々なネットワーク関連の操作を行うフロントエンドコマンドとして ip というコマンドが用意されていることがあります。
このコマンドを使っても route と同様の機能を利用することが可能です。その場合には ip コマンドの第一引数に route と指定します。
ただし、引数の書式は必ずしもrouteコマンドと同じではないので、注意しましょう。
例えば、「-net」「-host」といった引数は使えません。

・ipコマンド
ipコマンドは、ifconfigなどの旧来のネットワークコマンドに代わるコマンドです。
ipコマンドは様々な用途に使用できますが、ここでは基礎的なインターフェイスの設定やルーティングの設定について記載します。

以下はipコマンドの書式です。

ip オブジェクト オブジェクトに必要な構文

ここでは、最も基本的な例として、addressオブジェクトとrouteオブジェクトの例をとりあげます。
なお、オブジェクト名は省略した入力が可能で、addressオブジェクトは「addr」や「a」のように指定できます。

ip address コマンドの書式

アドレス表示
ip address show(list) [インターフェイス名]

アドレス設定 追加・削除
ip address { add | del } 宛先アドレス/プレフィックス dev インターフェース名

次の例は、最初にeth0の情報を表示し、IPアドレスを確認後に削除、再度別のIPアドレスを設定する例です。


ip route コマンドの書式

ルート表示
ip route [show(list)]

ルート設定 追加
ip route add 宛先アドレス via ゲートウェイのアドレス

ルート設定 削除
ip route del 宛先アドレス [via ゲートウェイのアドレス]
routeコマンド同様、削除の場合は、識別が可能な場合は宛先アドレスだけでの削除も可能です。

追加・削除の場合の宛先アドレスは、以下の形式が可能です。
IPアドレス/プレフィックス

デフォルトゲートウェイの追加と削除
ip route { add | del } default via デフォルトゲートウェイのアドレス
または
ip route del default

以下は、現状のルートを表示し、192.168.100.0/24へのゲートウェイを172.16.0.10と設定、また、デフォルトゲートウェイを172.16.0.100と設定し、表示した例です。その後、設定を逆順に削除しています。


・マルチホーム
マルチホームとは、同一のホストにおいて、複数のインターフェイスや複数のIPアドレスなどにより、ネットワークへ接続する経路が複数ある状態を言います。
クリティカルな業務を行うネットワークの場合、冗長性を持たせるために、マルチホームの構成になっていることは少なくありません。

マルチホームが、外部への接続における冗長性の確保のために用いられた場合、経路として、デフォルトゲートウェイが使われる場合があります。
デ フォルトゲートウェイのIPアドレスとして、外部と接続したルータのIPアドレスを指定しておけば、内部のネットワーク宛以外のパケット（ルーティング テーブルに定められた宛先の情報が無いパケット）は、すべてそのルータに送られ、そのルータを通して外部と接続できます。
ただし、デフォルトゲー トウェイは、同一のホストで2つ同時に動作すると、ルーティングテーブル上どちらに送信するか判断が困難になり得ますので、マルチホームの2つのネット ワークが両方ともデフォルトゲートウェイの設定を持つ場合は、なんらかの経路選択の方法が必要になります。
（なお、マルチホームのどちらのネット ワークから通信してどこと通信しても通信に支障のない構成や使用方法である場合、あえて同等のデフォルトゲートウェイを2つ持たせて負荷分散を図るような ネットワークも論理的には考えられますが、戻り通信の経路や通信の整合性を考えると、一般的な方法とは言えません。）
