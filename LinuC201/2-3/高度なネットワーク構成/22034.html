ヒット 2021年06月30日(水) 08時23分
ローカルネットワークを設定中で、1 分ごとに 192.168.1.2 への疎通を確認したい。
このような目的で利用することができるコマンドとして最も適切なものはどれか、選びなさい。

正解

ping -i 60 192.168.1.2

解説

ネットワークの設定を行う際には、指定のホストまで正しくパケットが伝送されるか、パケット往復にどれくらいの時間がかかるか等をチェックしたいという状況がよくあります。
そのような場合に利用できるコマンドとして ping があります。
ping コマンドは、引数に通信先のホストを指定して実行すると、そのホストに向けて ICMP パケットと呼ばれるパケットを送り、その結果を表示します。

書式
ping [オプション] ホスト

「ホスト」部はホスト名と IP アドレスどちらでも指定することが可能です。
また利用できるオプションとして主要なものは以下の通りです。


以上により、1 分ごとに疎通を確認するコマンドとして適切なものは
・ping -i 60 192.168.1.2
で、これが正解です。
このように -i オプションは後ろに数値を秒単位で指定すると、その間隔で疎通の確認 (ICMP パケットの送出) を行います。

残りの選択肢について見ていきます。

・ping -s 60 192.168.1.2
ping コマンドを利用しているところまでは正しいのですが、送信間隔の指定に -s オプションを使ってしまっている点が誤りです。
-s オプションはパケットのデータサイズを指定するオプションなので、このコマンドですとデータサイズを 60 バイトにするという意味になってしまいます。

・netstat -i 60 192.168.1.2
・netstat -s 60 192.168.1.2
netstat コマンドは自ホストの現在のネットワーク接続状態やルーティングテーブルの情報、パケット数の統計など、ネットワークに関した様々な情報を表示するためのコマンドです。
ホストを指定して疎通を確認するという機能はないため、誤りです。
なお netstat に -i や -s というオプションは存在しますが、まったく違う意味を持ったオプションですので、この選択肢のような指定は不正です。
