システムが保持している、同じネットワーク内の IP アドレスと MAC アドレスの対応関係を調べることができるコマンドとして、最も適切なものはどれか（2つ選択）

正解

arp

ip neigh show

解説

同じネットワーク内のIP アドレスと MAC アドレスの対応関係は、ARP キャッシュというテーブルとしてシステム内に保持されています（ARP キャッシュについての詳細は解説下部の「参考」をご覧下さい）。
ARP キャッシュに関する操作を行うコマンドは arp です。

書式
arp [オプション]

主要なオプションは以下の通りです。


「システムが保持している IP アドレスと MAC アドレスの対応関係を調べる」というのは ARP キャッシュの内容を参照することです。

実際に arp コマンドを実行した例を載せておきます。-n オプションを指定しているのは、逆引き（IP アドレスからホスト名を調べること）を行わず「Address」部に IP アドレスが表示されるようにするためです。


また、ARP キャッシュの内容は「ip neigh show」コマンドでも確認できます。

参考：
ネットワーク通信は、レイヤ（階層）構造を成しているプロトコルが、それぞれ独立した役割を果たすことによって実現されています。
例えば、物理的に直接接続されているデバイス間での通信方法を定義するレイヤ、ネットワーク上で離れた場所にあるホストと通信するための方法を定義するレイヤ、アプリケーションレベルで意味のある情報をやりとりする方法を定義するレイヤなどに分けられます。
普段ネットワークを利用している際に意識されるのは IP アドレスを用いた通信のレベルまでですが、そのような便利な通信が可能なのは、より下位のレイヤが物理的なデバイス間でのデータ転送などを制御してくれているためです。

IP アドレスを使う通信よりも下位のレイヤの通信に使うアドレスとして、MAC アドレスがあります。
これは、それぞれの物理的なデバイスに一意に（被らないように）書き込まれているアドレスです。
MAC アドレスにより物理的にそれぞれのデバイスを識別することができるので、直接接続されている（同じネットワーク内の）デバイス間ではこの MAC アドレスを使って通信を行うことができます。

しかしユーザは IP アドレスにより通信先を指定してくるわけですから、何らかの方法で IP アドレスとそれに対応する MAC アドレスを変換できなければなりません。
そのための方法（プロトコル）が ARP (Address Resolution Protocol) です。
ARP では MAC アドレスに変換したい IP アドレスを含む情報（パケット）をネットワーク内に一斉送信（ブロードキャスト）し、その IP アドレスに該当するホストは自分の MAC アドレスを含む情報（パケット）を送り返す、ということをします。
MAC アドレスを知らない相手と通信するためには、全体に問いかければいいという理屈です。

そして、以上の方法で IP アドレスと MAC アドレスの対応関係が判明すれば、それを手元（ARP キャッシュ）にメモします。
この ARP キャッシュにより、次からはいちいちブロードキャストを行わなくても、IP アドレスと MAC アドレスの相互変換が可能になるわけです。
arp コマンドは、この ARP キャッシュを操作・表示するためのコマンドなのです。
