次回起動に指定したカーネルの実ファイルを消去して、システムを起動した。その際の説明として適切なものは次のうちどれか。

正解

ブートローダがエラーを表示してシステムは停止する

解説

一般的な Linux システムでは、BIOS、ブートローダ、カーネル、init の順にブートプロセスが進行します。

カーネルをロードするのはブートローダですので、カーネルを見つけられない場合はブートローダがエラーメッセージを出します。

したがって正解は
・ブートローダがエラーを表示してシステムは停止する
です。

その他の選択肢については以下の通りです。
・init がエラーを表示してシステムは停止する
カーネルが起動した後に init が起動しますので、誤りです。

・bash がエラーを表示してシステムは停止する
カーネルが起動してOSが動作しないとbashを起動できませんので、誤りです。

・BIOS がエラーを表示してシステムは停止する
BIOSはカーネルではなくブートローダをロードしますので、誤りです。

・システムは通常に起動する
カーネルがないとシステムは起動しませんので、誤りです。

参考

一般的な Linux システムでは、BIOS、ブートローダ、カーネル、init の順にブートプロセスが進行します。

それぞれの役割と、各段階の詳細を以下に示しておきます。
この知識は、OS 起動に失敗する場合の問題特定の際にも重要ですので、自分で順を追って説明できるようにしておきましょう。

・BIOS
最もハードウェアに近い部分を司るシステムで、物理的なハードウェア（マザーボード）上に書き込まれています。
コンピュータの電源を入れると、まずこの BIOS が起動し、記憶装置（HDD）等に関して最低限の認識をした後、MBR（記憶装置の先頭セクタ、ブート用の特殊領域。ブートローダが格納されている） を読み込み、得られたブートローダに制御を移します。
なお、最近では BIOS に代わる UEFI （Unified Extensible Firmware Interface）が使用される場合があります。

・ブートローダ
Linux システムでは GRUB が該当します。
MBR に格納されている第一段階部分と、記憶装置内の別の場所に格納されている第二段階部分があります。
このように分かれている理由は、MBR には厳しいサイズ制限があるためです。
ブートローダは記憶装置内のカーネルをロードし、制御を移す役割を果たします。

・カーネル
カーネルは起動されると、高度にハードウェアを認識・制御し、ルートパーティションのマウントなど様々な初期化処理を行います。
その後、init という特別な最初のプロセスを起動します。

・init
最初に起動されるプロセスで、PID（プロセス ID）は必ず 1 です。
設定ファイル /etc/inittab の記述に基づいて、自動起動するべきプロセスを立ちあげるなど、アプリケーションレベルの初期化を行います。
以降、init プロセスは全てのプロセスの先祖（直接・間接的な呼び出し元）として存在し続けます。
なお、最近のシステムではinitプログラムとして、初期化処理を高速化したUpstartやsystemdを採用している場合があります。その場合は基本的に「/etc/inittab」ファイルは使用されません。
