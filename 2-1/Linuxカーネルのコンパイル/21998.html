ヒット 2021年06月28日(月) 18時39分
カーネルのソースを取得し、ビルドを行いシステムのカーネルを再構築を行った。しかし、ビルドが正常に終了し、新しいカーネルでブートが完了したものの、あるコマンドを実行し忘れたために必要なモジュールをロードする事ができなかった。
実行し忘れたと考えられるコマンドとして、最も適切なものを選びなさい。

正解

make modules_install

解説

ソースからのカーネルのビルドとインストール（再構築）の基本的な手順は以下の通りです。

カーネルのソースは取得・展開済みであるとし、以下はすべてソースを展開したディレクトリ直下で行います。

1. 設定を初期化
make mrproper で設定ファイルを含めてディレクトリ内を初期化します。
展開したソースにあらかじめ不適切な設定が含まれている可能性や、古いカーネルの設定が残っている可能性を考慮し、はじめに必ず実行するようにします。

2. 設定を行う
初期化された設定を今回の再構築用に設定します。設定はカーネルをコンパイル（ビルド）する前に行ないます。
この問題では「現在のカーネル設定を引き継」いで使うとのことなので、make oldconfig が適します。
他には make config で対話的に設定する方法、make menuconfig でコンソール上のメニューから設定する方法、make xconfig として X 上の GUIで設定する方法などがあります。

3. ビルドを行う
単に make とするか、make all とすることで、依存関係の解消、カーネル本体とモジュールのビルド、一時ファイルの削除を自動的に行ってくれます。

4. システムにインストールする
ビルドを行っただけではシステムに配置されてはいないので、インストールを行います。通常、make modules_install としてモジュールをインストールしてから、make install としてカーネル本体をインストールします。
make install によって /boot 以下にカーネルイメージが配置され、ブートローダに新しいカーネルを使う起動設定が追加されます。

以上の手順、および設問の「必要なモジュールをロードする事ができなかった」という記述を照らし合わせて考えると、make modules_install としてモジュールのインストールを忘れていたと考えられ、これが正解です。

他の選択肢についても見てみます。

・make install
カーネル本体をインストールし、ブートローダに新しいカーネルを使う起動設定を追加するコマンドです。
「新しいカーネルでブートが完了した」とありますので、このコマンドは実行されています。

・make clean
一時ファイル等を削除するコマンドです。
ビルドの成否に影響を及ぼす可能性はありますが、「ビルドが正常に終了した」とありますのでこの場合は関係がありません。

・mkinitrd
・mkinitramfs
初期 RAM ディスクに関するコマンドです。
初期 RAM ディスクに関係した問題で起動に失敗していれば疑うことができますが、そうではないので関係ないものと判断します。
