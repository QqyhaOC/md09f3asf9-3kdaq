説明

このシナリオでは、特定の少数のIPアドレスから数百万の不適切なリクエストを受信していることから、このアプリケーションはDDoS攻撃を受けていることがわかります。AWSリソースに対するDDoS攻撃を防御する際はAmazon Shieldを利用するのが一般的ですが、今回は迅速な改善策が求められています。したがって、まずは該当するIPアドレスからのアクセスをシャットアウトすることが必要です。

ある会社では、東京リージョンに Amazon EC2 インスタンスベース Webアプリケーションを実行しています。この EC2インスタンスは、複数のアベイラビリティーゾーン (AZ) に展開されており、Application Load Balancer (ALB) の背後にある Auto Scaling グループ内で起動されています。このシステムのデータベース層では、 プライベートサブネット内の EC2 インスタンスにMySQL データベースを構成しています。同社はシンガポールリージョンとソウルリージョンにもアプリケーションを展開することになりました。あなたは、複数のリージョンからの低レイテンシーなデータ読み込みを実現する構成が求められています。


最小の管理オーバーヘッドで、これらの要件を満たすことができる２つのソリューションはどれでしょうか。（2つ選択してください。）

説明

このシナリオでは、東京リージョンにホストされているAmazon EC2インスタンスベースの Webアプリケーションをシンガポールリージョンとソウルリージョンに展開することが必要です。その際に、複数のリージョンから低レイテンシーなデータ読み込みのみを実現する構成が求められています。現在は、プライベートサブネット内の EC2 インスタンス上にMySQL データベースを実行しているため、可用性が高い構成とはいえません。したがって、複数リージョンに展開可能なデータベースに移行することが必要となります。


オプション１が正解となります。EC2インスタンスベースのMySQLデータベースをAmazon Aurora MySQLデータベースに移行します。さらに、ソウルリージョンとシンガポールリージョンにAuroraレプリカを展開することで、各リージョンからの読み取りアクセスを最適化することができます。その際は、各リージョンのリーダーエンドポイントを利用してデータベースにアクセスする構成とします。これによって、ソウルリージョンとシンガポールリージョンはそれぞれのリージョンに設置されたAuroraレプリカからデータを直接読み込むことができるため、プライマリーDBにアクセスするよりも、低レイテンシーなデータアクセスが可能となります。

Auroraは標準的な RDS MySQL と比べて 5 倍のスループット、標準的な PostgreSQL と比べて 3 倍のスループットを実現することができます。Auroraはクラスターが3つのAZに分散されて構成されており、初めから高い可用性を担保することが可能です。さらにAmazon AuroraはAuroraレプリカ（リードレプリカ）を15台まで配置して、読み込み処理を分散化して、レポーティングなどの処理をマスターDBの処理と分けて、実効性を担保することができます。



オプション３も正解となります。Amazon Aurora Global Databaseは書き込み処理も含めて、Aurora DB全体を複数リージョンに展開するための仕組みであり、複数のリージョンからの低レイテンシーなデータ読み込みを実現することができます。

Amazon Aurora Global Database はDBクラスターが複数の AWS リージョン に配置されます。これにより、特定のリージョンからの低レイテンシーのグローバル読み取りを実現し、AWS リージョン 全体に影響が及ぶ可能性のある停止が発生しても、迅速に復旧します。Aurora Global Database には、1 つのリージョンにプライマリ DB クラスターがあり、異なるリージョンに最大 5 つのセカンダリ DB クラスターがあります。



オプション２と４は不正解です。Amazon RDSではグローバルDBを展開することができません。 Global DatabaseはAmazon Aurora のクラスターに適用可能な機能となっています。

オプション５は不正解です。Amazon Aurora Global Databaseは各リージョンのライターエンドポイントを利用してデータベースにアクセスすることができません。
