問題1: 正解

あるセキュリティ管理者は、高い読み取りスピードと耐障害性に重点を置いたRAID構成を作成する必要があります。複数のドライブが同時に故障することはまずありません。管理者が使用すべきRAID構成は次のうちどれですか。

説明

RAID（Redundant Array of Independent Disks）構成は、複数の物理ハードディスクを1つの論理ボリュームにまとめ、性能向上、耐障害性、またはその両方の組み合わせなど、さまざまな目的を達成するために使用されます。


セキュリティ管理者は、高い読み取り速度と耐障害性を優先したRAID構成を必要としています。複数のドライブが故障することはまずないので、管理者はパフォーマンスと冗長性のバランスが取れたRAIDレベルを選択することができます。


RAID 0：この構成では、冗長性を持たずに複数のディスクにデータをストライピングします。データが複数のドライブに分割されるため、高速な読み取りと書き込みが可能ですが、耐障害性はありません。1台のディスクが故障した場合、RAIDアレイ上のすべてのデータが失われます。


RAID 1： この構成では、2台のディスク間でデータをミラーリングします。すべてのデータは両方のディスクに同時に書き込まれ、フォールトトレランスを提供します。しかし、すべてのデータを2回書き込む必要があるため、性能上のメリットはありません。


RAID 5：この構成では、複数のディスク間でデータをストライプ化し、パリティ情報を使用してフォールトトレランスを提供します。1つのディスクが故障した場合、残りのディスクのパリティ情報を使用して、欠落したデータを再構築することができます。パフォーマンスと冗長性のバランスがとれています。


RAID 10：この構成は、RAID 1とRAID 0を組み合わせたものです。2組のストライプディスク間でデータをミラーリングし、高い読み取り速度とフォールトトレランスを提供します。少なくとも4台のディスクが必要で、各セットの1台のディスクの障害に耐えることができます。


要件を考慮すると、セキュリティ管理者は RAID 10 を選択する必要があります。RAID 10 は、高い読み取り速度とフォールトトレランスの両方を提供し、各セットの 1 つのディスクの障害に耐えることができます。RAID 5 もフォールトトレランスとパフォーマンス上の利点がありますが、リビルド操作中に 2 台目のディスクが故障するとデータが失われる可能性があるため、RAID 10 より信頼性は低くなります。

説明

CISOは、このリスクを軽減するために、ホットサイトフェイルオーバーの導入を最もおすすめする可能性が高いです。


ホットサイトフェイルオーバーとは、災害や障害発生時に迅速に活性化できるバックアップ施設のことです。通常、ビジネスの運用を再開するために必要なすべてのインフラストラクチャ、システム、データが含まれています。ホットサイトフェイルオーバーを導入することで、プライマリデータセンターが攻撃を受けたり、DDoS攻撃によるリソース消費が発生した場合でも、組織はこのセカンダリサイトに運用を切り替えることができます。


データセンターへの帯域幅のアップグレードは、ネットワーク容量を増やすことでDDoS攻撃に対する一定の軽減効果をもたらすかもしれません。ただし、攻撃中のデータベースリソースの消費に完全に対処するわけではありません。


顧客への完全なSaaS（Software-as-a-Service）提供への切り替えは、ローカルデータセンターへのDDoS攻撃の影響を軽減する可能性があります。ただし、組織の具体的な要件や制約によっては、実現可能な解決策とは限りません。


エンドユーザークエリに対してチャレンジレスポンステストを実装することは、DDoS攻撃によるデータベースリソースの消費の特定の懸念に直接関係していません。ユーザークエリの検証によってセキュリティが向上するかもしれませんが、DDoS攻撃のリソース消費には対処していません。

説明

対話型プロセスの悪用とは、システムの root 権限を得るために悪用を実行できるコマンドラインのことを指し、これは特権エスカレーションが該当します。


特権エスカレーションにはいくつかの方法があります。

一つは、既存の特権を使用して、より良い認証情報のセットを盗むアクションを実行する方法です。スニッファーを使って認証情報を取得したり、Windows セキュリティ・アカウント・マネージャ（SAM）や Linux/Unix etc/passwd ファイルを取得することで、上位の認証情報を得ることができます。

もう一つの方法は、エスカレートした特権で実行されているプロセスの脆弱性や弱点を突くことです。これらのプロセスに悪意のあるコードを注入することでも、特権への昇格を実現できます。


標的型攻撃とは、攻撃者が明確な意志と目的を持ち、その目的を達成するために執拗かつ継続的に攻撃を行います。


バッファ オーバーフローは、攻撃者が悪意のあるコードを実行したり、システムの制御を取得したりするために、プログラム内のバッファをオーバーフローさせる特定の種類のソフトウェア脆弱性です。


ファーミングはサイバー攻撃の一種で、攻撃者が Web サイトのトラフィックを悪意のある Web サイトにリダイレクトし、ユーザーを欺いて機密情報を盗むことを目的としています。

未回答
セキュリティ アナリストは、コンテナ内で実行されている一部のアプリケーションで検出された重大な脆弱性を懸念しています。最良の修復戦略は次のうちどれですか。

説明

コンテナ内で実行されているアプリケーションで重大な脆弱性が発見された場合は、通常、ベース コンテナ イメージを更新し、環境を再デプロイすることをお勧めします。この戦略が最も効果的である理由は次のとおりです。


    1.コンテナイメージは、コンテナインスタンスを作成するための基盤として機能します。基本コンテナー イメージを更新すると、更新されたイメージに含まれるセキュリティ パッチまたは更新が、そのイメージから派生したすべてのコンテナーに確実に適用されます。

    2.環境を再デプロイすると、更新された基本イメージを使用して新しいコンテナ インスタンスが作成され、以前のバージョンに存在した脆弱性が排除されます。

    3.ベースイメージを更新すると複数のコンテナインスタンス間で簡単にレプリケーションできるため、この戦略は効率的でスケーラブルです。


「サーバーの定期的なパッチ適用スケジュールにコンテナーを含める。」 サーバーに定期的にパッチを適用することは重要ですが、コンテナは特定のアプリケーション環境をカプセル化するため、個別のパッチ適用が必要です。これらをサーバーのパッチ適用スケジュールに含めても、コンテナーに固有の脆弱性に対処できない可能性があります。

「実行中の各コンテナに個別にパッチを適用し、アプリケーションをテストする。」このアプローチは、特に多数のコンテナを扱う場合、時間がかかり、エラーが発生しやすくなります。基本コンテナー イメージを更新して環境を再デプロイする方が効率的です。

「コンテナーが実行されているホストを更新する。」 ホスト システムを最新の状態に保つことは不可欠ですが、ホストを更新するだけではコンテナー自体内の脆弱性に対処できない可能性があります。コンテナー化されたアプリケーションはホスト システムから分離できるため、ホストを更新してもコンテナー化されたアプリケーションは自動的には更新されません。

    1.資格情報のチェックイン/チェックアウト: PAM システムにより、特権資格情報の安全なチェックインとチェックアウトが可能になります。管理者は、必要に応じて認証情報へのアクセスをリクエストし、タスクが完了したら認証情報を返すことができます。これにより説明責任が確保され、不正アクセスのリスクが軽減されます。

    4.資格情報へのアクセスのログ記録: PAM システムは通常、詳細なログ記録機能と監査機能を提供します。特権認証情報へのアクセスを記録および追跡し、組織がセキュリティとコンプライアンスの目的でアクティビティを監視およびレビューできるようにします。

説明

安全でないプロトコルは、特定のシナリオにおけるデータ侵害の最も可能性の高い原因です。安全でないプロトコルとは、ユーザーのブラウザと企業の Web サーバー間の通信が暗号化されていないことを意味し、送信される情報は攻撃者によって傍受されやすい状態になります。


ユーザーが個人情報と秘密の質問への回答を提供すると、通常はインターネット経由で会社のサーバーに送信されます。 Web サイトが HTTP などの安全でないプロトコルを使用している場合、攻撃者はこの通信を傍受し、機密データを盗む可能性があります。


一方、入力検証が行われていないということは、Web サイトがユーザーから提供されたデータの正確性、完全性、安全性をチェックしていないことを意味します。これにより、SQL インジェクションやクロスサイト スクリプティング攻撃などのさまざまな問題が発生する可能性がありますが、セキュリティで保護されていないプロトコルよりもデータ侵害が発生する可能性は低くなります。


オープン権限とは、ユーザーまたはプロセスに必要以上のアクセス権を与えることを指し、不正アクセスやデータの悪用につながる可能性があります。これは懸念事項ではありますが、Web サイトと転送中のユーザー データのセキュリティには直接関係しません。


パッチが適用されていないということは、Web サイトのソフトウェアが最新のセキュリティ修正を適用されていないことを意味し、攻撃者が悪用できる脆弱性が生じる可能性があります。ただし、攻撃者はこれらの脆弱性を利用して機密データにアクセスする方法を見つける必要があるため、セキュリティで保護されていないプロトコルよりもデータ侵害が発生する可能性は低くなります。
